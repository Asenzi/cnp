# 位置同步机制说明

## 📍 更新策略

### 当前策略（已优化）

1. **应用启动 (onLaunch)**：必定同步位置
   - 用户打开小程序时
   - 冷启动时触发

2. **应用显示 (onShow)**：节流同步
   - 距离上次同步超过 5 分钟才会更新
   - 避免频繁切换时重复调用

3. **登录成功**：立即同步位置
   - 用户登录成功后立即获取并上传

### 节流机制

为了平衡实时性和性能，采用以下策略：
- ✅ **onLaunch**: 无节流，每次启动都更新
- ⏱️ **onShow**: 5分钟节流，避免频繁调用
- 🚀 **登录**: 无节流，立即更新

## 🎯 预期行为

- **首次打开小程序**: 获取位置并上传 ✅
- **切换到后台再返回（5分钟内）**: 不更新 ⏭️
- **切换到后台再返回（5分钟后）**: 重新更新 ✅
- **完全关闭后重新打开**: 必定更新 ✅
- **登录成功**: 立即更新 ✅

### 3. 常见问题检查清单

#### ✓ 小程序权限配置
检查 `manifest.json` 中是否包含：
```json
"mp-weixin": {
  "requiredPrivateInfos": ["getLocation", "chooseLocation"],
  "permission": {
    "scope.userLocation": {
      "desc": "用于展示您所在城市与附近人脉"
    }
  }
}
```

#### ✓ 用户授权状态
用户是否在小程序设置中授权了位置信息？
- 如果未授权，`uni.getLocation` 会失败
- 可以通过测试页面的按钮测试

#### ✓ 登录状态
位置同步仅在用户已登录时执行：
- 检查 `uni.getStorageSync('token')` 是否存在
- 未登录用户会在日志中看到"未登录"

#### ✓ 缓存机制
位置同步采用智能缓存，1小时内不会重复上传：
- 清除缓存后重新进入 app 应该会触发同步
- 可以在测试页面查看缓存时间

#### ✓ 后端 API
确认后端支持经纬度更新：
```python
# 后端应该接受 latitude 和 longitude 字段
PATCH /api/v1/user/me
{
  "latitude": 39.9042,
  "longitude": 116.4074
}
```

## 🛠️ 解决方案

### 方案 1: 清除缓存测试
```javascript
// 在控制台执行
uni.removeStorageSync('userLocation')
// 然后重启小程序
```

### 方案 2: 强制同步
如果需要立即同步而不管缓存，可以临时修改代码：
```javascript
// 在 App.vue onLaunch 中
syncUserLocation(true) // 使用 syncUserLocation 而不是 smartSyncLocation
```

### 方案 3: 检查网络请求
在开发者工具的 Network 面板中：
1. 查找 `/api/v1/user/me` 的 PATCH 请求
2. 检查 Request Payload 是否包含 latitude 和 longitude
3. 检查 Response 是否成功（200 状态码）
