# 🔄 重新编译小程序步骤

## 问题原因
控制台日志显示 "位置缓存仍然有效，跳过更新"，这是旧代码的输出。
新代码应该输出 "[位置同步] 开始同步位置（每次都更新）"

## ✅ 解决方案

### 方案 1: 完全重新编译（推荐）

1. **停止开发服务器**
   - 在终端中按 `Ctrl + C` 停止当前运行

2. **清理构建产物**
   ```bash
   # 在项目 front 目录下执行
   rm -rf unpackage/dist/*
   # 或者手动删除 unpackage/dist 文件夹
   ```

3. **重新启动**
   ```bash
   npm run dev:mp-weixin
   ```

4. **在微信开发者工具中**
   - 点击 "编译" 按钮（或按 Ctrl+B）
   - 等待编译完成

### 方案 2: 微信开发者工具重新编译

1. **点击顶部菜单**：项目 -> 重新构建 npm
2. **点击工具栏**："编译" 按钮
3. **清除缓存**：
   - 点击 "清缓存"
   - 选择 "清除全部缓存"
4. **重新编译**

### 方案 3: 强制刷新

在微信开发者工具中：
- Windows: `Ctrl + Shift + R`
- Mac: `Cmd + Shift + R`

## 🔍 验证是否生效

重新编译后，打开控制台应该看到：

```
[App] onLaunch - 检查登录状态: 已登录
[App] onLaunch - 用户已登录，立即同步位置
[位置同步] 开始同步位置（每次都更新）  ← 看到这个就对了！
[位置同步] 开始同步用户位置，静默模式: true
[位置同步] 开始获取用户位置...
```

## ⚠️ 如果还是不行

1. **确认文件已保存**
   - 检查编辑器标签页是否有 "●" 标记
   - 按 `Ctrl + S` 保存所有文件

2. **重启微信开发者工具**
   - 完全关闭开发者工具
   - 重新打开项目

3. **检查是否修改了正确的文件**
   - 确认修改的是 `front/utils/location-sync.js`
   - 而不是 `unpackage` 目录下的编译后文件
