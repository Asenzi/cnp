# 感兴趣功能实现说明

## 功能概述
实现了用户对其他用户的"感兴趣"功能，支持标记和取消标记，状态持久化到数据库。

## 后端修改

### 1. 数据库模型
**新增文件**: `backend/app/models/user_interest.py`
- 创建 `UserInterest` 模型
- 字段：user_pk（感兴趣的用户）、target_user_pk（被感兴趣的用户）
- 唯一约束：防止重复标记

**修改文件**: `backend/app/models/__init__.py`
- 注册 `UserInterest` 模型

### 2. CRUD操作
**修改文件**: `backend/app/crud/network.py`
- 新增 `toggle_user_interest()`: 切换感兴趣状态
- 新增 `get_user_interests()`: 批量查询感兴趣状态

### 3. 推荐服务
**修改文件**: `backend/app/network/recommend_service.py`
- 在 `list_network_recommendations()` 中：
  - 调用 `get_user_interests()` 获取感兴趣状态
  - 在返回的每个用户数据中添加 `is_interested` 字段

### 4. API端点
**修改文件**: `backend/app/api/v1/network.py`
- 新增 `POST /api/v1/network/interest/toggle`: 切换感兴趣状态
  - 参数：target_user_id（目标用户的business_user_id）
  - 返回：{ is_interested: boolean, message: string }

### 5. 数据库迁移
**新增文件**: `backend/migrations/create_user_interests_table.sql`
- 创建 `user_interests` 表
- 创建索引
- 创建自动更新时间戳的触发器

## 前端修改

### 1. API封装
**修改文件**: `front/api/network.js`
- 新增 `toggleUserInterest(targetUserId)`: 调用后端切换感兴趣API

### 2. 页面逻辑
**修改文件**: `front/pages/tab/discover/index.vue`
- 修改 `onInterestMember()` 函数：
  - 调用后端API而不是仅更新前端状态
  - 根据后端返回的状态更新UI
  - 保留反馈事件用于推荐算法

### 3. UI组件
**修改文件**: `front/pages/tab/discover/components/NetworkMemberCard.vue`
- 已支持根据 `member.interested` 状态显示不同样式
- 未感兴趣：灰色空心 ♡
- 已感兴趣：红色实心 ♥

## 使用流程

1. **用户点击感兴趣按钮**
   - 前端调用 `toggleUserInterest(targetUserId)`
   - 后端切换数据库中的感兴趣状态
   - 返回新状态给前端

2. **刷新页面**
   - 前端调用 `getNetworkRecommendations()`
   - 后端查询感兴趣状态并在返回数据中包含 `is_interested` 字段
   - 前端根据 `is_interested` 显示正确的按钮状态

3. **数据持久化**
   - 感兴趣关系存储在 `user_interests` 表
   - 支持查询、切换、删除操作

## 数据库迁移步骤

```bash
# 进入后端目录
cd backend

# 执行迁移脚本
sqlite3 your_database.db < migrations/create_user_interests_table.sql

# 或者如果使用其他数据库，需要相应调整SQL语法
```

## 测试要点

1. ✅ 点击感兴趣按钮，状态立即更新
2. ✅ 再次点击，取消感兴趣
3. ✅ 刷新页面，状态保持
4. ✅ 不同用户之间的感兴趣状态独立
5. ✅ 不能对自己标记感兴趣
6. ✅ 同一用户对同一目标只能有一条记录

## 注意事项

1. 需要先执行数据库迁移脚本创建 `user_interests` 表
2. 后端API需要认证，确保用户已登录
3. 前端需要处理API调用失败的情况
4. 感兴趣状态会影响推荐算法（通过反馈事件）
